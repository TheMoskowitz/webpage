Deep Learning Projects



Three projects I built as part of my self-study in Deep Learning

Project #1 is an app that can describe your surroundings for the visually impaired. The architecture used is basically the same as that proposed in the original 'show-and-tell' paper (a CNN-LSTM model for image captioning) but with a pre-trained Inception as the CNN.

The most difficult part of this project was getting it to work in Android after I discovered, after having already spent almost two weeks training it on the Flickr30k database, that one of the operations was not supported on Android at all. I eventually figured out how to edit and remove nodes from a Tensorflow graph, which is a bit of a hack as you are only meant to be able to add nodes to Tensorflow graphs, not take them away. I wrote a guide on how to do this, which is on github.

Project #2 is a vision system for an autonomous vehicle. The neural network categorizes every pixel in its incoming video feed as a) Road, b) Sidewalk, c) Vehicle (including bicycles), d) Person or Other.

The model I ended up using was a hybrid of a Unet and a Fully Convolutional Net and I trained it on both the KITTI dataset and then later on the Cityscapes dataset (the final model in the video is trained on Cityscapes). All of the code for each of the variations I experimented with and more detail on their pros and cons is available in the project's github repository.

Project #3 takes raw music audio as its input and transcribes that audio into sheet music. This was the most difficult of the three as it strayed the farthest from what other researchers have done before. It also required the development of a completely new CNN architecture that's better suited for operating on the Short-Time Fourier Transform of the music data and a great deal of creative data augmentation.




Robo-Trump



Our project for GeekCon 2016 - Robo-Trump!

We built this over the course of a long weekend. Robo-Trump is controlled by an app and has two tasks. The first task is Greet the Voters. When Greeting the Voters, Robo-Trump looks around for faces and when he finds one, he checks the person's skin color and then insults him or her with increasing severity depending on how dark the person's skin is (so as to resemble the real Donald Trump).

Robo-Trump's other task is Speech. If you choose this task then the robot gives an impassioned speech about inferior quality Chinese and Mexican robots stealing the jobs of their hardworking American counterparts.

We also went to some lengths to mimic Trump's distinctive hand gestures (he's a fully-functional robotic torso with arms that bend at the shoulder and elbow as well as servos at the neck and waist). His movements were programmed with the ROSS system.

I was responsible for Robo-Trump's control app, his text-to-speech and his computer vision system (plus all of his insults). My teammates Ziv Barcesat, Oren Meiri, Gil Meiri and Omer Koren and I were quite proud of our final robotic candidate, though admittedly he was only ~85% finished by the end of the Hackathon.

We have made the code available on github but it comes with a pretty severe WARNING. This is Hackathon code, put together with no sleep and at breakneck speed. It is NOT readable, dependable production code -- it is held together with duct tape and silly putty. Some of it probably doesn't work and whatever is uploaded may not have been the final product. Please judge accordingly.




Rufus the Personal Assistant



This is a project I built to control the various electronic devices in my room. The android app uses bluetooth to connect to the arduino which then, through a system of relays, controls the power flowing into each outlet of the surge protector. I keep each device I want to control plugged into the surge protector and can then turn them on and off from the app.

The ingredients are a smartphone, an arduino uno, relays, a bluetooth module (HC-05) and a store-bought surge protector. If you are trying to replicate this, make sure to buy a surge protector that you can open with regular screwdrivers. Many of them are specifically designed to prevent you getting in and rewiring them. And, this should go without saying, but be very sure which wires are which when you do open it up. This will be plugged into the wall and wiring it wrong can do serious damage. Definitely have someone else double check what you're doing. Finally make sure to use electrician grade wire (the very thick kind). If you use typical hobby wires with this project (or any that involves mains power) you are setting yourself up for disaster and may well be putting yourself in danger.

I've been using this thing for almost a year now and in that time I've made some updates to the code that you may find useful. A quirk of bluetooth connection protocol is that there is no automatic programmatic notice when you are out of range. One way to solve this problem would be to ping the bluetooth device every ten seconds to make sure the connection is still valid but that seemed particularly wasteful to me. So instead, this app only performs the check when a button is pressed. If you press a button, the app will set a timer and if it doesn't receive a response in that time (a second or two) it will attempt to reconnect. If the reconnect fails you will know you're not connected as the borders of the buttons will turn off. Otherwise it will get the current on/off status of each of the outlets and display them to you.

You can find the code for both the android app and the arduino on my github page.




The Nintendophone



The Nintendophone is a musical instrument that can be played the way you would a video game. The name Nintendophone is a combination of the gaming company "Nintendo" and the musical suffix "phone" (think saxophone or xylophone).

It was built with an arduino, a sparkfun musical instrument shield, a switch, a joystick, some black perspex and a bunch of buttons. It's based around scales, which you can cycle through with the yellow button (at the moment it includes blues, major, minor and a spanish phrygian scale I like). The notes are played with the joystick. I've given it a sort of dirty, electric trumpet sound. You can also use the other buttons to change the octave and key signature.

The Nintendophone also has two special features: flipping the switch on the side turns it into a drumset and if you hit all the buttons at the same time, it plays the Super Mario Brothers Theme.

If I were to build it again, I would probably include an on/off switch and volume control. Right now I just plug it in to turn it on and I use the amp volume control, but it would be handy if I had it at my fingertips on the device itself. I would also probably switch out the drumset which I don't use that often for an 8-bit sound.

You can find the Nintendophone's code on my github page.




The Talking Box



Why does the Talking Box exist? It's a mystery.

I built it with an arduino, a servo, a really cool emic 2 chip I had lying around and a wine box a friend was storing at my house (sorry Lee). I had to play with the code for awhile to get it to open and close its mouth at an appropriate speed, with semi-natural pauses between sentences. Initially I thought I would have it read out my emails to me in the morning but upon further testing, I concluded it was too annoying for that purpose. So now I'm open to suggestions.
